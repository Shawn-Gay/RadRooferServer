---
name: review
description: Review recently changed code for quality, security, and tenant-isolation issues before committing.
model: opus
---

Review all changes since the last commit using: !`git diff HEAD`

Also run: !`git status`

Check for each of the following and report findings as **CRITICAL**, **WARNING**, or **SUGGESTION** with `file:line` references:

**Tenant Isolation**
- Any EF Core query missing the global query filter (manual `.Where(o => o.TenantId == ...)` instead of relying on filter)
- `IgnoreQueryFilters()` called outside of seed/admin/migration code
- EF entity returned directly in API response instead of mapped to a DTO

**Security**
- Missing `[Authorize]` on a controller or action that touches tenant data
- `[AllowAnonymous]` added to a route that isn't login or a webhook
- Vapi webhook payload used before `X-Vapi-Secret` is verified
- Secrets, connection strings, or API keys in source code
- String comparison on secrets (should use `CryptographicOperations.FixedTimeEquals`)

**Async Correctness**
- `.Result` or `.Wait()` on a Task
- `async void` method
- `SaveChanges()` (sync) instead of `SaveChangesAsync()`

**EF Core**
- `CreatedAt` or `UpdatedAt` set manually in a handler or constructor
- `Id` set manually (should be generated by PostgreSQL)
- Missing `.AsNoTracking()` on a read-only query
- `.Include()` on a write path where the nav property isn't needed for writing back

**C# Style**
- `var` used where the type isn't obvious from the right-hand side
- Nested ternary operators
- Sync-over-async patterns

Output a summary at the end: how many CRITICAL / WARNING / SUGGESTION findings, and whether it is safe to commit.
